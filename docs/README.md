# Documentation

Design documentation for OpenMesh Messenger. The authoritative requirements are
in the top-level [`SRS.md`](../SRS.md); everything here elaborates *how* those
requirements are met.

- [`architecture/`](architecture/) — component and layer design (SRS §2, §10–§12).
- [`protocol/`](protocol/) — the wire format and packet reference (SRS §8, §9).
- [`crypto/`](crypto/) — key, session and encryption design (SRS §5, §6).
- [`adr/`](adr/) — Architecture Decision Records: one file per significant
  decision, newest numbered highest.
